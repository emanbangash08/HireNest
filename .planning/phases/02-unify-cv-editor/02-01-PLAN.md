<objective>
Refactor `CvEditorPanel` to use the existing `CvDocumentRenderer` (structured JsonResume editor) as the primary editing surface, falling back to the freeform `InPlaceCvEditor` only for legacy/non-standard CVs.

Purpose: Eliminate the brittle freeform renderer as the default editing experience. The freeform editor depends on `__vh_tags` which are often missing or incorrect for odd CV structures, causing broken layouts. The structured `CvDocumentRenderer` already supports all standard JsonResume sections with proper form fields.
Output: A single `CvEditorPanel` component that intelligently chooses the best editor based on CV data shape.
</objective>
