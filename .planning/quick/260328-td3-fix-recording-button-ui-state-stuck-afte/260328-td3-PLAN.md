# Quick Task: Fix Recording Button UI State Stuck After Release

**Problem:** After releasing the hold button, the recording button UI state gets stuck in the "isListening" state (showing REC with pulse animation) even though recording has stopped.

**Root Cause:** In `electron/src/App.tsx`, the `setTranscriptState` callback function (line 94-98) calls `setTranscript(text)` on line 95, but `setTranscript` is not defined in App.tsx. The transcript state is managed by the `useAudioRecording` hook and cannot be directly set from App.tsx. This causes a runtime error that prevents the UI state from updating correctly.

**Solution:** Remove the invalid `setTranscriptState` callback and its usages. The transcript state should only be managed by the `useAudioRecording` hook.
