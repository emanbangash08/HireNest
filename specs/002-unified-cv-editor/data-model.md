# Data Model: Unified Editable A4 CV Workspace

## Entity: CVDocument (existing)

- **Purpose**: Persisted CV content and metadata used across management, review, ATS, and generation.
- **Core Fields**:
  - `id: string`
  - `userId: string`
  - `cvJson: JsonResumeSchema`
  - `templateId: string`
  - `isPrimary: boolean`
  - `jobApplicationId?: string | null`
  - `createdAt: datetime`
  - `updatedAt: datetime`
- **Validation Rules**:
  - `userId` must match authenticated user for read/write.
  - `templateId` must be an available template ID.
  - `cvJson` must be valid against app-level JSON Resume schema expectations.

## Entity: EditableA4Session (new logical entity)

- **Purpose**: Runtime representation of the single A4 page editing state.
- **Core Fields**:
  - `sessionId: string`
  - `cvId: string`
  - `variantType: enum(primary, branch, job-specific)`
  - `template: PdfmeTemplate`
  - `inputs: PdfmeInputs` (single-page input object)
  - `saveStatus: enum(idle, unsaved, saving, saved, error)`
  - `lastSavedAt?: datetime`
  - `version: number` (optimistic concurrency)
- **Validation Rules**:
  - `cvId` must exist and belong to the current user.
  - `template` and `inputs` must remain structurally compatible.
  - `version` must increment after each successful save.

## Entity: PdfTemplateBinding (new logical entity)

- **Purpose**: Deterministic mapping between `cvJson` and `pdfme` template inputs for in-place A4 editing.
- **Core Fields**:
  - `bindingId: string`
  - `templateId: string`
  - `fieldMap: Record<string, CvJsonPath>`
  - `normalizers?: Record<string, string>`
  - `updatedAt: datetime`
- **Validation Rules**:
  - Every required template field must map to a valid source path or have a documented default.
  - Mapping output must produce a valid `inputs[0]` object for the template.

## Entity: PdfExportArtifact (existing/new logical representation)

- **Purpose**: Downloadable PDF generated from the same template and active session inputs.
- **Core Fields**:
  - `artifactId: string`
  - `cvId: string`
  - `variantType: enum(primary, branch, job-specific)`
  - `sourceSessionVersion: number`
  - `mimeType: "application/pdf"`
  - `createdAt: datetime`
- **Validation Rules**:
  - Export source must match the active sessionâ€™s `cvId` and `version`.
  - Generated bytes must be derived from `template + inputs` currently used by editable session.

## Entity: MigrationRun (new)

- **Purpose**: Track one-time migration execution from legacy editing scheme to unified A4 workspace scheme.
- **Core Fields**:
  - `runId: string`
  - `startedAt: datetime`
  - `completedAt?: datetime`
  - `status: enum(running, completed, failed)`
  - `processedCount: number`
  - `migratedCount: number`
  - `skippedCount: number`
  - `failedCount: number`
  - `failureLogPath?: string`
- **Validation Rules**:
  - Run summary counts must be internally consistent.
  - Migration script must be idempotent for safe re-runs.

## Entity: LegacyCvEditingState (deprecated)

- **Purpose**: Represent historical split edit/preview state and compatibility fields targeted for removal.
- **Core Fields**:
  - `legacyViewMode?: string`
  - `legacyDraftPreviewState?: object`
  - `legacyTemplateRenderState?: object`
- **Validation Rules**:
  - Must not be used by runtime flows after cutover.
  - Must be transformed or removed by migration script.

## Relationships

- `CVDocument (1) -> (0..n) EditableA4Session`
- `CVDocument (n) -> (1) PdfTemplateBinding` via `templateId`
- `EditableA4Session (1) -> (0..n) PdfExportArtifact`
- `MigrationRun (1) -> (0..n) CVDocument` processed records

## State Transitions

### EditableA4Session Save State

1. `idle -> unsaved` when user edits any field on A4 surface.
2. `unsaved -> saving` when debounce timer or explicit save triggers.
3. `saving -> saved` when backend save succeeds and returns incremented version.
4. `saving -> error` when save fails.
5. `error -> saving` when user retries or next autosave attempt triggers.
6. `saved -> unsaved` on subsequent edit.

### Variant Context Safety

1. Variant selected (`primary`/`branch`/`job-specific`).
2. Session bound to selected `cvId`.
3. Save/export operations validate active `cvId` before execution.
4. If mismatch or stale version detected, operation fails with conflict response and reconciliation prompt.

### Migration and Cutover Safety

1. Migration script reads legacy CV editing records.
2. Script generates unified workspace-compatible state and removes deprecated legacy fields.
3. Script records per-run migration metrics in `MigrationRun`.
4. Application runtime uses only new workspace schema after cutover.
